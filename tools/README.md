# tools

`agent-friend` 的**工具与脚本集合**。

## 定位

存放服务于"开发 / 调试 / 评估"的代码，**不属于产品本身**。

孵化期的典型用例：

- **调试 UI**：与 `agent` 进行多轮对话的本地入口（CLI / 简易 Web，待 [`001/design.md`](../docs/requirements/001-foundation-chat-and-memory/) 决定）
- **eval 脚本**：批量验证记忆系统在固定场景下的表现（如"教 AI 一个事实 → 重启 → 召回测试"）
- **数据导入 / 导出**：未来可能需要的迁移脚本

未来可能的扩展：

- 性能 benchmark
- 记忆数据库的查看器 / dump 工具
- 与 LLM Provider 的连通性测试

## 状态

孵化期实现（至少要有调试 UI）。

## 内部结构

无强制约束。每个工具/脚本可以是一个子目录，自带 README 说明用法。

```
src/tools/
├── debug_ui/       # 调试 UI（孵化期实现）
├── eval/           # eval 脚本（按需）
└── ...
```

## 与其他模块的依赖

- 依赖 `agent`（调用对话能力做调试）
- 依赖 `memory`（查看 / 操作记忆数据）
- 依赖 `llm_providers`（连通性测试等）
- 不被任何其他模块反向依赖
